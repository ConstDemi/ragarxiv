# Журнал экспериментов

Что пробовали, что прижилось, что выкинули. Подробности каждого рана (per-question дампы,
точные промпты, карточки) — в MLflow.

## Сетап

- **Golden:** 50 QA-пар (25 Q1 «существует ли статья о X» / 25 Q2 «как решают проблему X»).
- **Метрики:** RAGAS — `faithfulness`, `answer_relevancy`, `context_precision`, `context_recall`;
  плюс `doc_recall@context_k` / `@retrieved` (детерминированный, без судьи).
- **Судья:** Claude Haiku. ~$2 за прогон (n=50).
- **«Линейка» (ruler `vN`):** чем меряем (golden / судья / что судья видит). Меняется → раны несравнимы.
  Текущая **v2** = golden-50 + судья видит только `context_k` чанков (что реально видела LLM).
- **Шум при n=50:** ~±0.05 — меньшие дельты не значимы. Сравниваем **парно** (по вопросам), а не средние
  (иначе таймаут-NaN маскируется под изменение).

## Журнал (линейка v2)

| Run | Что меняли | Результат | Вердикт |
|---|---|---|---|
| `gen-3b-v2` | базлайн 3B | faithfulness 0.705 | базлайн для сравнения |
| `gen-7b-v2` | генератор 3B → 7B (оба 4-bit) | faithfulness 0.705 → **0.833** | ✅ **KEPT** (дефолт) |
| `prompt-grounded-v2` | +4 правила блоком в системный промпт | faithfulness 0.833 → 0.696, relevancy 0.719 → 0.640, англ-ответов 9 → 17 | ❌ **REVERT** (регресс) |
| `prompt-lang-v2` | +1 клауза «отвечать на языке вопроса» | relevancy +0.005 (шум), англ-ответов 9 → 8 | ❌ **REVERT** (ноль) |

> Более ранние раны `gen-3b-v0` (pilot-golden) и `gen-3b-v1` (golden-50 до `context_k`) лежат в MLflow,
> но это **другие линейки** — с v2 напрямую не сравниваются.

## Выводы

- **Генератор решает.** 3B → 7B дал +0.13 faithfulness — крупнейший выигрыш. 7B стал дефолтом.
- **Промпт-ось на 7B исчерпана.** Блок правил → регресс (over-refusal + дрейф языка); одиночная клауза → ноль.
  7B плохо слушает нюансы системного промпта — baseline остаётся оптимумом. Правки промпта больше не ведём.
- **Ретрив — не узкое место.** `doc_recall` 0.96/0.98; из 11 провалов faithfulness 10 имели нужный документ
  в контексте → потеря в **генерации**, не в поиске. Поэтому reranker/сильный эмбеддер отложены (Фаза 2).
- **Удешевление оценки.** Судить по `context_k` чанкам (а не всем retrieved) дешевле и корректнее
  ($3.5 → ~$2); `ContextPrecision` — главный драйвер цены (вызов на каждый чанк).

## Как читать раны в MLflow

- Имя: `<axis>-<variant>-<ruler>` (напр. `prompt-lang-v2`).
- Теги: `axis` / `variant` / `ruler` / `compare_to` / `status` (`kept` / `reverted` / `superseded`).
- Артефакты: `system_prompt.txt` (точный промпт рана) + `per_question_*.parquet` (по-вопросные оценки).
- Поле **Description** — карточка: Итог / Изменение / База / Гипотеза / Ожидания.
